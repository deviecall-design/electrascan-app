# Changelog

All notable changes to ElectraScan are documented here.
Format based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/).

## [1.1.1.0] - 2026-05-08

### Added

- **Rate library CRUD** — Add rate, edit rate, and delete rate operations are now fully wired to Supabase. All three mutations update the UI immediately (optimistic update) and roll back on error with a toast message.
- `deleteRate(id)` added to `services/supabaseData.ts` — deletes a row from `rate_library` by ID.
- Add/edit modal in Pricing Schedule screen: code, category, description, unit, material rate, labour rate fields with live total preview.
- **DocuSign "Send for Approval" real integration** — the button in the Approvals screen now POSTs to `/api/docusign/envelope` with signer name, email, estimate ref, project name, and a placeholder PDF. On success, shows the envelope ID in a green banner and logs to `approval_audit`. On failure, shows an error toast without advancing the sent state.
- Editable signer name and email fields on the Approvals screen (pre-filled from the first signatory).
- 13 new tests: `rateLibraryCrud.test.ts` (deleteRate + upsertRate) and `docusignSend.test.ts` (6 integration scenarios covering success, error, audit insert, network failure).

### Changed

- Approvals screen: `Send for Approval` button now shows loading spinner during API call and is disabled until the call resolves.

## [1.1.0.0] - 2026-05-07

### Added

**Desktop workflow (6-screen app)**
- Full desktop shell with Anthropic-sourced design system (Poppins/Lora/JetBrains Mono, warm-cream palette, orange CTA convention)
- Dashboard screen with live Supabase KPI strip: estimates this month, pipeline value, win rate, avg scan-to-quote
- Detection screen + 4-step scan detail flow: upload → AI scan → review queue → approve
- Estimate screen with sequential reference numbering (EST-YYYY-NNN-rev)
- Pricing Schedule screen with per-tenant rate library
- Variation Report screen: delta-against-prior-estimate workflow
- Approvals screen: DocuSign multi-party signing workflow (Phase 1 wiring)
- Settings screen: company profile CRUD — name, ABN, phone, email, logo URL, address; persists to Supabase `company_profile` table with live/draft status indicator

**Multi-tenant schema**
- `tenants` and `tenant_memberships` tables with owner/member roles and RLS
- `get_tenant_id()` helper function for RLS policy use
- `tenant_id` FK added to estimates, scans, rate_library, variations, timesheets, milestone_claims
- Seed row for Vesh Electrical (tenant #0)

**Symbol intelligence**
- `symbol_dictionary` table: per-entity learned symbol-to-SKU mappings with confidence scoring
- `legend_library` table: builder/architect legend definitions from pricing schedule scans
- `upsert_symbol_classification()` RPC for tradie-confirmed symbol classifications
- `lib/symbol_map.ts`: 400+ AS/NZS electrical symbol mappings

**Data layer**
- `variation_reports` table with tenant-scoped RLS
- `company_profile` table with owner-scoped RLS
- `useSupabaseQuery` hook: live Supabase data with graceful mock fallback
- Live KPI queries: estimates this month, pending pipeline value, win rate, avg scan-to-quote
- Dashboard auth filtering: all KPI queries scoped to `owner_id = auth.uid()`
- Atomic estimate reference counter (replaces COUNT-based race condition)
- Legend library table for builder/architect symbol mapping

**PDF exports**
- Branded estimate PDFs with company logo, ABN, and contact details
- Variation report PDF generation

### Fixed

- Detection pipeline: legend items now always receive a price (no more 0-component scans)
- Dashboard KPI queries: added `owner_id` auth filter to prevent cross-user data exposure
- Estimates: switched `total` → `value` column to match actual schema
- Symbol map: renamed SWITCHING_DIMMER key from `SD` to `SWD` to fix silent overwrite by SMOKE_DETECTOR (duplicate key TS1117)
- `upsert_symbol_classification` SECURITY DEFINER RPC: added `auth.uid()` ownership guard to prevent cross-user symbol dictionary poisoning
- Reference sequence: replaced COUNT-based sequence with atomic counter table to eliminate race condition on concurrent estimate creation

### Changed

- App entry point switched from mobile-first `App.tsx` to desktop-first `DesktopApp.tsx` via `react-router-dom` v6
- Tenant ID writes stamped on all estimate and variation report inserts
